# Project agent rules

- Read `Instructions.md` before working on the narrative implementation.
- Preserve existing architectural conventions unless there is a documented
  reason to change them.
- Dialogue content and narrative state should remain data-driven where practical.
- Do not alter story canon without recording the change.
- Run the relevant automated tests after implementation.
- Validate both endings manually or through an appropriate gameplay test harness.
- Never claim visual or gameplay validation based only on compilation.
